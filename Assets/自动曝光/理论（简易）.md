Unity URP 自动曝光技术实现文档 (基于 Compute Shader)
文档版本说明：本技术文档基于 Unity 2022.3.62 (URP 14) 编写。核心算法来源于 Unity 6000 版本，已完整降级并适配 2022 的 CommandBuffer 与 RTHandle 架构，供后续复习与项目迭代使用。

第一章：技术原理概述
自动曝光的目的是模拟人眼或真实相机的适应机制：在明亮环境中自动压暗画面，在黑暗环境中自动提亮画面。

1.1 摄影学曝光值 (EV100)
在图形学中，我们不直接用“亮度百分比”来控制曝光，而是使用物理单位 EV100（基于 ISO 100 的曝光值）。

公式：EV = log2(Luminance * K)，此处 K 取 8.0（不同引擎有微调）。
转换乘数：最终要乘在屏幕颜色上的系数为 Exposure = 1.0 / pow(2.0, EV)。EV 越大，画面越暗。
1.2 屏幕空间亮度统计与权重
要计算当前画面的 EV，需要先算出“平均亮度”。但直接算术平均是不对的：

人眼关注中心：使用到屏幕中心距离的平方衰减 pow(1.0 - distance, 2) 作为权重，忽略画面边缘的纯黑或过曝。
对数空间：人眼对光的感知是对数的。因此，在计算局部曝光时，先将线性亮度转为 log2(Luminance)，在对数空间进行模糊计算，结果更符合视觉规律。
原子操作：在 Compute Shader 中，成千上万个线程同时写同一个变量会导致数据丢失。必须使用 InterlockedAdd 进行原子累加，并将浮点数放大 100 倍转为整数（GPU 对整数原子操作更快/更兼容）。
1.3 时间滤波
如果每帧直接应用当前画面的 EV，遇到明暗交替时画面会疯狂闪烁。引入 historyEV，每帧通过 deltaTime 进行线性插值（LERP），实现平滑的“人眼适应过程”。

1.4 局部曝光与保边滤波
全局曝光的痛点：逆光看人（人脸极暗，背景极亮），全局 EV 会因为背景拉高，导致人脸死黑。

解决方案：算出每个像素“周围局部区域”的 EV，与全局 EV 做差值，单独提亮暗部。
双边滤波：普通的均值/高斯模糊会把背景的亮光“晕染”到人脸边缘，产生光晕。双边滤波在计算权重时，不仅考虑空间距离（离得越远权重越小），还考虑亮度差异（如果邻居像素亮度差很大，说明是边缘，权重强制拉低）。从而实现“内部平滑，边缘锐利”。
分离式高斯模糊：O(N^2) 的 2D 模糊极其耗时。利用高斯函数的可分离性，拆分为“横向 1D + 纵向 1D”，计算量降为 O(2N)。
第二章：C# 管线架构搭建 (Unity 2022.3)
在 URP 中，自定义后处理需要三个层级：VolumeComponent (参数) -> ScriptableRendererFeature (注入器) -> ScriptableRenderPass (执行者)。

2.1 参数声明：AutoExposure.cs
继承 VolumeComponent，将参数暴露给美术。

csharp

using System;
using UnityEngine.Rendering;

[Serializable]
[VolumeComponentMenu("Custom/AutoExposure")]
public class AutoExposure : VolumeComponent
{
    // 控制是否开启局部曝光（对应HLSL的变体开关）
    public BoolParameter localExposure = new BoolParameter(false);
    // 控制双边滤波与高斯模糊的混合度
    public ClampedFloatParameter blurredLumBlend = new ClampedFloatParameter(0.5f, 0f, 1f);
}
2.2 管线注入器：RendererFeature
负责将 Pass 挂载到 URP 管线。这里以局部曝光为例，全局曝光结构相同。

csharp

using UnityEngine;
using UnityEngine.Rendering.Universal;

public class LocalExposureRendererFeature : ScriptableRendererFeature
{
    public LocalExposureRenderSettings localExposureRenderSettings;
    private LocalExposureRenderPass localExposureRenderPass;

    public override void Create()
    {
        localExposureRenderPass = new LocalExposureRenderPass(localExposureRenderSettings);
    }

    public override void AddRenderPasses(ScriptableRenderer renderer, ref RenderingData renderingData)
    {
        if (localExposureRenderSettings.computeShader == null) return;
        renderer.EnqueuePass(localExposureRenderPass);
    }

    // 【重点】2022.3 必须手动释放 GPU 资源
    protected override void Dispose(bool disposing)
    {
        localExposureRenderPass?.Dispose();
    }
}
2.3 核心执行者：LocalExposureRenderPass.cs (适配 2022.3)
这是最复杂的部分。由于没有 6000 的 Render Graph 自动管理内存，我们必须手动管理 RTHandle（中间纹理）和 ComputeBuffer。

关键点：

[StructLayout(LayoutKind.Sequential)]：强制 C# 结构体内存顺序与 HLSL 完全一致，防止 GPU 读取乱码。
RenderingUtils.ReAllocateIfNeeded：2022.3 安全分配带 UAV（随机写入）权限的纹理的推荐做法。
动态分支剔除：如果 Volume 关闭了局部曝光，直接跳过耗时的模糊 Kernel。

第三章：GPU 计算
使用 Compute Shader 将整个屏幕拆分为 16x16 的线程组进行并行计算。

3.2 Kernel 1: AccumulateLuminance (统计与降维)
全屏并行，将 1920x1080 的像素信息压缩成两个整数。

hlsl

[numthreads(16, 16, 1)]
void AccumulateLuminance(uint3 id : SV_DispatchThreadID, uint groupIndex : SV_GroupIndex)
{
    // 单线程清零初始化
    if (!any(id.xy)) { _RWParameters[0].importance = 0; _RWParameters[0].luminance = 0; }
    
    // 1. 计算中心权重
    half2 uv = id.xy / _ScreenSize.xy;
    float importance = pow(saturate(1.0 - (distance(uv, 0.5) * 2.0)), 2);

    // 2. 计算感知亮度 (BT.709 标准)
    float luminance = dot(_ScreenTexture[id.xy].rgb, float3(0.2126, 0.7152, 0.0722));
    
    // 3. 整数量化 + 原子累加
    uint imp = (uint)(100.0 * importance);
    uint lum = (uint)(100.0 * luminance * importance);
    InterlockedAdd(_RWParameters[0].importance, imp);
    InterlockedAdd(_RWParameters[0].luminance, lum);
    
    // 4. 输出对数亮度供局部曝光使用
    _LogLuminance[id.xy] = log2(max(luminance, 1e-5));
}
3.3 Kernel 2: ComputeTargetEV (单线程决策)
hlsl

[numthreads(1, 1, 1)]
void ComputeTargetEV(uint3 id : SV_DispatchThreadID)
{
    // 还原并计算加权平均亮度
    float avgLuminance = (0.01 * _RWParameters[0].luminance) / max(0.01 * _RWParameters[0].importance, 1e-5);
    
    // 计算当前帧 EV
    float avgEV = log2(max(avgLuminance, 1e-5) * 8.0);
    
    // 时间平滑防闪烁 (限制最大步长防止卡顿跳变)
    float t = min(_RParameters[0].deltaTime, 0.033);
    float targetEV = _RWParameters[0].historyEV + (avgEV - _RWParameters[0].historyEV) * t;
    
    // 状态更新
    _RWParameters[0].historyEV = targetEV;
    _RWParameters[0].exposure = 1.0 / pow(2.0, targetEV); // 转为乘数
}
3.4 Kernel 3: BilateralBlur (保边平滑)
hlsl

[numthreads(16, 16, 1)]
void BilateralBlur(uint3 id : SV_DispatchThreadID)
{
    float centerLum = _LogLuminance[id.xy];
    float sumLum = 0, sumWeight = 0;
    
    // 7x7 邻域采样
    for (int y = -3; y <= 3; y++) {
        for (int x = -3; x <= 3; x++) {
            int2 coord = clamp(id.xy + int2(x, y), 0, _ScreenSize.xy - 1);
            float sampleLum = _LogLuminance[coord];
            
            // 空间高斯权重
            float wSpatial = exp(-((x*x + y*y) / (2 * 2.0 * 2.0)));
            // 亮度范围高斯权重 (差异大->权重趋近0->保护边缘)
            float wRange = exp(-((sampleLum - centerLum)*(sampleLum - centerLum)) / (2 * 0.3 * 0.3));
            
            float w = wSpatial * wRange;
            sumLum += sampleLum * w;
            sumWeight += w;
        }
    }
    _BilateralLogLuminance[id.xy] = sumLum / max(sumWeight, 1e-5);
}
3.5 Kernel 4: GaussianBlur (分离式高性能模糊)
hlsl

[numthreads(16, 16, 1)]
void GaussianBlur(uint3 id : SV_DispatchThreadID)
{
    // Pass 1: 水平模糊，从 _LogLuminance 读，写入 _GaussianTempBuffer
    float sumH = 0, weightH = 0;
    for (int x = -3; x <= 3; x++) {
        int sx = clamp(id.x + x, 0, (int)_ScreenSize.x - 1);
        float w = exp(-(x*x) / (2.0 * 2.0 * 2.0));
        sumH += _LogLuminance[int2(sx, id.y)] * w;
        weightH += w;
    }
    _GaussianTempBuffer[id.xy] = sumH / max(weightH, 1e-5);

    // Pass 2: 垂直模糊，从 _GaussianTempBuffer 读，写入 _GaussianLogLuminance
    float sumV = 0, weightV = 0;
    for (int y = -3; y <= 3; y++) {
        int sy = clamp(id.y + y, 0, (int)_ScreenSize.y - 1);
        float w = exp(-(y*y) / (2.0 * 2.0 * 2.0));
        sumV += _GaussianTempBuffer[int2(id.x, sy)] * w;
        weightV += w;
    }
    _GaussianLogLuminance[id.xy] = sumV / max(weightV, 1e-5);
}
3.6 Kernel 5: ApplyExposure (最终合成)
hlsl

[numthreads(16, 16, 1)]
void ApplyExposure(uint3 id : SV_DispatchThreadID)
{
    float3 color = _ScreenTexture[id.xy].rgb;
    float globalExposure = _RWParameters[0].exposure;
    float localExposure = 1.0;

    #ifdef LOCAL_EXPOSURE
        // 将对数空间还原回线性空间
        float bilateralLum = exp2(_BilateralLogLuminance[id.xy]);
        float gaussianLum = exp2(_GaussianLogLuminance[id.xy]);
        
        // 混合两种模糊结果
        float localLum = lerp(bilateralLum, gaussianLum, _RParameters[0].blurredLumBlend);
        
        // 计算局部与全局的 EV 差距
        float localEV = log2(max(localLum, 1e-5) * 8.0);
        float diffEV = localEV - _RWParameters[0].historyEV;
        
        // 计算局部乘数 (乘 0.5 是为了保留一定明暗对比，不要完全拉平)
        localExposure = pow(2.0, -diffEV * 0.5);
    #endif

    // 叠加全局与局部曝光
    _ScreenTexture[id.xy] = float4(color * globalExposure * localExposure, 1.0);
}
第四章：踩坑记录与排错指南
UAV 写入报错 (Silver)
现象：Compute Shader 写入 _ScreenTexture[id.xy] = xxx 时，画面全黑或报错。
原因：URP 默认分配的相机目标纹理没有开启 enableRandomWrite。
解决：在 C# 端 OnCameraSetup 中克隆描述符，强制设置 descriptor.enableRandomWrite = true;，并用 Blit 拷贝到这张新图上处理。
C# 与 HLSL 数据错位
现象：HLSL 里的 historyEV 读出来是一堆乱码小数。
原因：C# 的 struct 默认内存对齐方式与 GPU 的 cbuffer / StructuredBuffer 不同。
解决：C# 结构体上方必须加 [StructLayout(LayoutKind.Sequential)]，且使用 Marshal.SizeOf 计算字节大小。
局部曝光产生奇怪的光晕/反色
现象：画面边缘出现不正常的发光。
原因：在 C# 端 ApplyExposure 绑定纹理时，参数写反了（例如把 _GaussianLogLuminance 绑给了 bilateralLum 的槽位）。
解决：严格检查 cmd.SetComputeTextureParam 的字符串与 HLSL 声明是否一一对应。
渲染管线断层 (黑屏)
现象：执行完 Compute 后，屏幕什么都看不见。
原因：算完的图存在自己声明的 RTHandle 里，URP 不知道。
解决：最后一定要 cmd.Blit(screenTexture, renderer.cameraColorTargetHandle) 把图还给管线。
切分辨率或编辑器失焦后显存泄漏/报错
现象：拉大 Game 窗口后报错。
原因：2022.3 不会自动清理 RTHandle。
解决：使用 RenderingUtils.ReAllocateIfNeeded 自动处理分辨率变化，并在 RendererFeature.Dispose 中彻底 Release()。
