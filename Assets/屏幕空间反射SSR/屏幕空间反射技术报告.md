# URP 屏幕空间反射 SSR 技术报告

## 1. 实现目标

本目录实现的是一个用于 Unity URP 的屏幕空间反射 Screen Space Reflection, SSR 后处理效果。

SSR 的核心思想是：利用当前相机已经渲染出来的屏幕颜色、深度和法线信息，在屏幕空间内为每个像素发射一条反射射线，寻找这条射线和深度缓冲中可见表面的交点。如果找到交点，就从交点对应的屏幕颜色中采样反射颜色，再叠加回当前相机颜色。

当前实现包含以下能力：

- URP `ScriptableRendererFeature` 注入自定义 SSR 渲染 pass。
- 请求 URP 生成深度纹理和法线纹理。
- 复制当前相机颜色，避免读写同一个 color target。
- 构建 HiZ 分层深度纹理，用更粗的 mip 层加速射线步进。
- 在 shader 中进行屏幕光栅空间 DDA raymarching。
- 使用二分搜索细化命中 UV。
- 使用 4x4 dither 打散起始步进，减少条带感。
- 对 SSR 结果做横向和纵向高斯模糊。
- 支持 Additive 和 Balance 两种合成方式。

## 2. 相关文件

### `shader 1/SSR.cs`

这是 SSR 的 URP RendererFeature 和 RenderPass 主脚本，负责：

- 暴露 SSR 参数到 RendererData Inspector。
- 创建 `Hidden/SSR` 材质。
- 判断 shader pass 数量是否满足当前渲染流程。
- 请求深度和法线输入。
- 分配临时 RTHandle。
- 生成 HiZ 深度金字塔。
- 按顺序执行 SSR、模糊和合成 pass。

### `shader 1/SSR.shader`

这是 SSR 的 fullscreen shader，包含 5 个 pass：

1. `Generate HiZ`
2. `Raymarching`
3. `Blur`
4. `Additive`
5. `Balance`

它负责：

- 从深度重建世界空间位置。
- 从法线纹理读取屏幕像素法线。
- 计算反射方向。
- 将视空间射线投影到屏幕像素坐标。
- 在屏幕空间步进并做深度相交测试。
- 使用 HiZ mip 层跳步。
- 命中后做二分 refinement。
- 计算边缘淡出、距离淡出和 Fresnel 权重。
- 输出反射颜色和 alpha 权重。
- 模糊反射纹理。
- 将反射结果合成回相机颜色。

### `shader 1/DepthNormalsFeature.cs`

这是一个辅助 RendererFeature。它的作用是通过 `ConfigureInput(ScriptableRenderPassInput.Normal)` 请求 URP 生成 `_CameraNormalsTexture`。

当前 `SSR.cs` 自己已经在 `Setup` 中请求了：

```csharp
ConfigureInput(ScriptableRenderPassInput.Depth | ScriptableRenderPassInput.Normal);
```

所以 SSR 激活时已经会要求 URP 准备深度和法线。`DepthNormalsFeature.cs` 仍然可以作为备用或调试用 Feature：当项目中某些管线配置没有稳定生成法线纹理时，可以挂载它来单独触发 DepthNormals prepass。

### `屏幕空间反射_Renderer.asset`

这是当前 URP RendererData。它已经挂载了名为 `SSR` 的 RendererFeature，当前关键配置为：

```yaml
m_IntermediateTextureMode: 1
mSettings:
  Intensity: 0.8
  MaxDistance: 10
  Stride: 8
  StepCount: 64
  Thickness: 0.25
  BinaryCount: 5
  jitterDither: 1
  UseHiZ: 1
  MipCount: 8
  blendMode: 0
  BlurRadius: 1
  RayBias: 0.03
  DistanceFade: 2
  EdgeFade: 8
  FresnelStrength: 0.35
```

其中 `m_IntermediateTextureMode: 1` 很重要。SSR 需要读取并写回相机颜色，使用中间颜色纹理可以避免直接操作 backbuffer 时无法读取或 RTHandle 为空的问题。

## 3. 整体渲染流水线

当前 SSR 的完整执行顺序如下：

1. URP 渲染不透明物体。
2. SSR RendererFeature 在 `AfterRenderingOpaques` 注入。
3. RenderPass 通过 `ConfigureInput` 请求 `_CameraDepthTexture` 和 `_CameraNormalsTexture`。
4. `OnCameraSetup` 根据相机分辨率分配临时纹理。
5. 如果开启 `UseHiZ`，分配 HiZ 深度金字塔 RT。
6. C# 将 SSR 参数写入 shader。
7. `Execute` 中先构建 HiZ。
8. 复制当前相机颜色到 `_SSRSourceCopy`。
9. 执行 `Raymarching` pass，输出 SSR 反射结果到 `_SSRTexture0`。
10. 执行横向 blur，将 `_SSRTexture0` 输出到 `_SSRTexture1`。
11. 执行纵向 blur，将 `_SSRTexture1` 输出回 `_SSRTexture0`。
12. 执行最终合成 pass，将 `_SSRTexture0` 与 `_SSRSourceCopy` 合成回相机颜色。

用数据流表示：

```text
Camera Color
    |
    | copy
    v
_SSRSourceCopy -------------------------------+
    |                                         |
    | Raymarching: depth + normal + HiZ       |
    v                                         |
_SSRTexture0                                  |
    |                                         |
    | Horizontal Blur                         |
    v                                         |
_SSRTexture1                                  |
    |                                         |
    | Vertical Blur                           |
    v                                         |
_SSRTexture0                                  |
    |                                         |
    | Additive / Balance Composite            |
    v                                         |
Camera Color <--------------------------------+
```

HiZ 数据流：

```text
Camera Depth
    |
    | Blit mip 0
    v
_SSRHiZBufferMip0
    |
    | Generate HiZ pass
    v
_SSRHiZBufferMip1
    |
    | Generate HiZ pass
    v
...
    |
    | CopyTexture each mip
    v
_SSRHiZBuffer, use as _HierarchicalZBufferTexture
```

## 4. `SSR.cs` 技术解释

### 4.1 参数结构 `SSRSettings`

```csharp
internal class SSRSettings {
    internal float Intensity = 0.8f;
    internal float MaxDistance = 10.0f;
    internal int Stride = 8;
    internal int StepCount = 64;
    internal float Thickness = 0.25f;
    internal int BinaryCount = 5;
    internal bool jitterDither = true;
    internal bool UseHiZ = true;
    internal int MipCount = 8;
    internal BlendMode blendMode = BlendMode.Addtive;
    internal float BlurRadius = 1.0f;
    internal float RayBias = 0.03f;
    internal float DistanceFade = 2.0f;
    internal float EdgeFade = 8.0f;
    internal float FresnelStrength = 0.35f;
}
```

各参数含义：

| 参数 | 作用 |
| --- | --- |
| `Intensity` | 反射强度。越大反射越明显。 |
| `MaxDistance` | 反射射线在视空间中最多走多远。 |
| `Stride` | 每次 raymarch 在屏幕像素上的步长。大步长更快但更容易漏交。 |
| `StepCount` | 最大步进次数。越大越稳定但越耗性能。 |
| `Thickness` | 深度相交容差，用来判断射线是否穿过屏幕表面。 |
| `BinaryCount` | 命中后进行二分搜索的次数。 |
| `jitterDither` | 是否使用 4x4 dither 打散起始步进。 |
| `UseHiZ` | 是否使用 HiZ 分层深度优化。 |
| `MipCount` | HiZ 深度金字塔 mip 数量。 |
| `blendMode` | 最终合成方式，Additive 或 Balance。 |
| `BlurRadius` | 反射结果模糊半径。 |
| `RayBias` | 起始点沿反射方向偏移，减少自反射。 |
| `DistanceFade` | 随步进距离衰减反射。 |
| `EdgeFade` | 屏幕边缘淡出，减少边缘断裂。 |
| `FresnelStrength` | Fresnel 权重强度，控制掠射角反射更强的程度。 |

### 4.2 RendererFeature 生命周期

`Create` 中创建自定义 RenderPass，并将注入点设置为：

```csharp
renderPassEvent = RenderPassEvent.AfterRenderingOpaques;
```

这样 SSR 会在不透明物体渲染完成后执行。此时深度、法线和相机颜色已经有内容，可以进行屏幕空间反射。

`AddRenderPasses` 中只允许 Game 和 SceneView 相机执行：

```csharp
CameraType cameraType = renderingData.cameraData.cameraType;
if (cameraType != CameraType.Game && cameraType != CameraType.SceneView)
    return;
```

这可以避免材质预览、Inspector 预览等特殊相机触发 SSR，降低空 RTHandle 或无效 pass 的风险。

随后通过 `GetMaterial` 查找 `Hidden/SSR` shader，并确保 shader 至少有 5 个 pass：

```csharp
return mMaterial != null && mMaterial.passCount >= RenderPass.RequiredPassCount;
```

如果 shader pass 不完整，SSR 不会入队，避免运行时出现 invalid pass index。

### 4.3 输入声明

RenderPass 的 `Setup` 中调用：

```csharp
ConfigureInput(ScriptableRenderPassInput.Depth | ScriptableRenderPassInput.Normal);
```

这告诉 URP：当前 pass 需要深度纹理和法线纹理。之后 shader 中可以使用：

```hlsl
SampleSceneDepth(uv)
SampleSceneNormals(uv)
```

读取屏幕深度和法线。

### 4.4 临时 RTHandle 分配

`OnCameraSetup` 使用当前相机的 `cameraTargetDescriptor` 作为基础描述：

```csharp
mDescriptor = renderingData.cameraData.cameraTargetDescriptor;
mDescriptor.msaaSamples = 1;
mDescriptor.depthBufferBits = 0;
```

SSR 临时纹理不需要深度缓冲，也不应该继承 MSAA。然后分配：

```csharp
_SSRSourceCopy
_SSRTexture0
_SSRTexture1
```

其中：

- `_SSRSourceCopy` 保存原始相机颜色。
- `_SSRTexture0` 保存 raymarching 或纵向 blur 后的 SSR 结果。
- `_SSRTexture1` 保存横向 blur 中间结果。

### 4.5 HiZ RT 分配

当 `UseHiZ` 开启时调用 `AllocateHiZTextures`。

HiZ 基础尺寸使用相机尺寸的下一个 2 的幂再除以 2：

```csharp
int width = Mathf.Max(1, Mathf.NextPowerOfTwo(mDescriptor.width) >> 1);
int height = Mathf.Max(1, Mathf.NextPowerOfTwo(mDescriptor.height) >> 1);
```

这样做的目的：

- HiZ mip 链需要连续缩小。
- 使用 2 的幂尺寸更适合生成完整 mip。
- 除以 2 可以减少 HiZ 成本。

最终分配一个带 mip 的 RFloat RT：

```csharp
new RenderTextureDescriptor(width, height, RenderTextureFormat.RFloat, 0) {
    useMipMap = true,
    autoGenerateMips = false,
    mipCount = mHiZMipCount,
    sRGB = false
};
```

`RFloat` 只存一个深度通道，足够表达深度金字塔。

另外还分配每一层 mip 的临时 RT：

```csharp
mHiZBufferMipTextures[i]
```

这些纹理用于逐级生成 mip，随后通过 `cmd.CopyTexture` 复制到真正的 `_SSRHiZBuffer` mip 链中。

### 4.6 向 shader 传参

`OnCameraSetup` 中设置：

```csharp
_SourceSize = (width, height, 1 / width, 1 / height)
_SSRParams0 = (MaxDistance, Stride, StepCount, Thickness)
_SSRParams1 = (BinaryCount, Intensity, 0, 0)
_SSRParams2 = (RayBias, DistanceFade, EdgeFade, FresnelStrength)
```

这些参数在 shader 中控制射线长度、步进、命中厚度、衰减和权重。

同时根据开关启用 shader keyword：

```csharp
_JITTER_ON
_HIZ_ON
```

keyword 的作用是让 shader 编译出不同分支，关闭相关功能时不执行对应逻辑。

### 4.7 Execute 的完整命令顺序

`Execute` 是实际渲染入口。

第一步检查材质、相机颜色和临时 RT 是否有效：

```csharp
if (mSourceCopyTexture == null || mSSRTexture0 == null || mSSRTexture1 == null || cameraColor == null || cameraColor.rt == null)
    return;
```

如果当前相机不是正常可写颜色 RT，就跳过 SSR。

第二步构建 HiZ：

```csharp
BuildHiZBuffer(cmd, renderingData.cameraData.renderer.cameraDepthTargetHandle);
cmd.SetGlobalTexture(_HierarchicalZBufferTexture, mHiZBufferTexture.nameID);
```

第三步复制相机颜色：

```csharp
Blitter.BlitCameraTexture(cmd, cameraColor, mSourceCopyTexture);
cmd.SetGlobalTexture(_SSRSourceTexture, mSourceCopyTexture.nameID);
```

复制是必要的，因为后面最终合成要写回 `cameraColor`。如果一边读 cameraColor 一边写 cameraColor，容易产生未定义结果。

第四步执行 SSR raymarching：

```csharp
Blitter.BlitCameraTexture(cmd, mSourceCopyTexture, mSSRTexture0, mMaterial, Raymarching);
```

第五步做横向和纵向 blur：

```csharp
_SSRBlurRadius = (BlurRadius, 0, 0, 0)
_SSRTexture0 -> _SSRTexture1

_SSRBlurRadius = (0, BlurRadius, 0, 0)
_SSRTexture1 -> _SSRTexture0
```

第六步合成回相机颜色：

```csharp
Blitter.BlitCameraTexture(cmd, mSSRTexture0, cameraColor, mMaterial, Addtive 或 Balance);
```

## 5. `SSR.shader` 技术解释

### 5.1 通用 include

shader 引入：

```hlsl
#include "Core.hlsl"
#include "DeclareDepthTexture.hlsl"
#include "DeclareNormalsTexture.hlsl"
#include "Blit.hlsl"
```

用途：

- `Core.hlsl` 提供矩阵、空间变换、URP 基础函数。
- `DeclareDepthTexture.hlsl` 提供 `SampleSceneDepth`。
- `DeclareNormalsTexture.hlsl` 提供 `SampleSceneNormals`。
- `Blit.hlsl` 提供 fullscreen triangle 的 `Vert` 和 `_BlitTexture`。

### 5.2 关键纹理

```hlsl
TEXTURE2D_X(_SSRSourceTexture);
TEXTURE2D_X(_HierarchicalZBufferTexture);
```

`_SSRSourceTexture` 是 C# 复制出来的原始相机颜色。Raymarch 命中后会从这里取反射颜色，最终合成时也从这里取原图颜色。

`_HierarchicalZBufferTexture` 是 C# 构建的 HiZ 深度金字塔。开启 `_HIZ_ON` 后，raymarching 会从不同 mip 层采样深度。

### 5.3 深度天空判断

```hlsl
bool IsSkyDepth(float rawDepth)
{
#if UNITY_REVERSED_Z
    return rawDepth <= 0.000001;
#else
    return rawDepth >= 0.999999;
#endif
}
```

Unity/URP 通常使用 reversed Z。reversed Z 下天空或无几何位置的深度接近 0。这个函数用来跳过天空像素，避免对无效深度做重建和反射。

### 5.4 4x4 Dither

```hlsl
float Dither4x4(float2 pixel)
```

它使用一个固定 4x4 阈值表，根据当前像素坐标返回 0 到 1 的 jitter 值。这个值只作用在 raymarch 的第一次步进：

```hlsl
float stepScale = (i == 0) ? jitter : 1.0;
```

作用是把所有像素相同的起始步长打散，减少 SSR 常见的条带状伪影。

### 5.5 视空间到屏幕空间投影

```hlsl
float4 ProjectViewToScreen(float3 positionVS)
```

该函数把视空间位置乘以投影矩阵，转成屏幕像素坐标：

```hlsl
positionCS = UNITY_MATRIX_P * positionVS
uv = positionCS.xy / positionCS.w
uv.y *= _ProjectionParams.x
uv = uv * 0.5 + 0.5
pixel = uv * _SourceSize.xy
```

SSR 后续不直接在 3D 空间固定距离步进，而是在屏幕像素坐标上做 DDA，这样更接近屏幕 rasterization 的行为。

### 5.6 二分搜索

`BinarySearchHitUV` 的输入是：

- 命中前一个点 `lowP / lowQ / lowK`
- 命中当前点 `highP / highQ / highK`
- 是否发生主轴交换 `permute`
- 二分次数 `binaryCount`

函数每次取中点，重新采样深度：

```hlsl
float rayZ = midQ.z / midK;
float sceneZ = -LinearEyeDepth(rawDepth, _ZBufferParams);
```

如果射线深度已经到表面之后，就把高点收缩到中点；否则把低点推进到中点。经过多次迭代后得到更接近真实交点的 UV。

二分搜索可以减少步长较大导致的命中点偏移，让反射采样更稳定。

### 5.7 HiZ 生成 pass

`FragGenerateHiZ` 从上一级深度 mip 采样 2x2 深度：

```hlsl
float4 depths = float4(
    sample(-0.5, -0.5),
    sample(-0.5,  0.5),
    sample( 0.5, -0.5),
    sample( 0.5,  0.5)
);
```

然后根据 Z 方向选择 min 或 max：

```hlsl
#if UNITY_REVERSED_Z
    depth = max(...)
#else
    depth = min(...)
#endif
```

在 reversed Z 下，越靠近相机深度值越大，所以 mip 中保留最大值。这样粗 mip 能表示这一块区域内最靠近相机的遮挡表面。

### 5.8 Raymarching pass

`FragRaymarching` 是 SSR 的核心。

第一步读取当前像素深度并重建世界坐标：

```hlsl
float rawDepth = SampleSceneDepth(uv);
float3 positionWS = ComputeWorldSpacePosition(uv, rawDepth, UNITY_MATRIX_I_VP);
```

第二步读取世界法线并计算视线方向和反射方向：

```hlsl
float3 normalWS = normalize(SampleSceneNormals(uv));
float3 viewDirWS = normalize(positionWS - _WorldSpaceCameraPos);
float3 rayDirWS = normalize(reflect(viewDirWS, normalWS));
```

第三步转到视空间：

```hlsl
float3 startVS = TransformWorldToView(positionWS);
float3 rayDirVS = normalize(TransformWorldToViewDir(rayDirWS));
```

视空间中相机朝向通常是负 z，所以如果反射方向不是朝屏幕内部走，就直接返回空反射：

```hlsl
if (rayDirVS.z >= -0.0001)
    return 0;
```

第四步计算起点和终点：

```hlsl
startVS += rayDirVS * RayBias;
endVS = startVS + rayDirVS * MaxDistance;
```

`RayBias` 用来避免射线刚出发就命中自己。

第五步将起点和终点投影到屏幕像素坐标：

```hlsl
startScreen = ProjectViewToScreen(startVS);
endScreen = ProjectViewToScreen(endVS);
```

第六步构造透视校正插值量：

```hlsl
k = 1 / w
q = viewPosition * k
```

屏幕空间线性插值时，不能直接线性插值 view z。需要插值 `q` 和 `k`，再用：

```hlsl
rayZ = q.z / k;
```

还原当前步进点的视空间 z。

第七步 DDA 主轴处理：

```hlsl
bool permute = abs(diff.x) < abs(diff.y);
```

如果 y 方向变化更大，就交换 x/y，让循环始终沿主轴递增。这样可以避免斜率太大导致采样不均匀。

第八步计算每次步进增量：

```hlsl
dp = pixel increment * Stride
dq = perspective corrected view increment * Stride
dk = inverse w increment * Stride
```

第九步循环 raymarch：

```hlsl
for (int i = 0; i < stepCount && p.x * dir <= end; i++)
```

每次步进后得到：

- 当前屏幕像素 `hitPixel`
- 当前 UV `hitUV`
- 射线深度范围 `rayZMin/rayZMax`
- 场景深度 `sceneZ`

核心相交判断为：

```hlsl
bool isBehindSurface = rayZMin + rayBias <= sceneZ;
bool intersectsSurface = isBehindSurface && rayZMax >= sceneZ - thickness;
```

含义是：射线当前段进入了屏幕表面之后，并且没有超过允许厚度太多，就认为命中。

### 5.9 HiZ raymarch 分支

开启 `_HIZ_ON` 时：

```hlsl
stepScale *= exp2(mipLevel);
hitRawDepth = SampleHierarchicalDepth(hitUV, mipLevel);
```

mip 越高，每次步进越大。没有进入表面时，提升 mip：

```hlsl
mipLevel = min(mipLevel + 1.0, maxMipLevel);
```

如果粗 mip 判断可能命中，则回退到上一个点并降低 mip：

```hlsl
p = prevP;
q = prevQ;
k = prevK;
mipLevel -= 1.0;
```

最终只有当 `mipLevel == 0` 时才进行精确命中和二分。这样可以用粗层快速跳过大片无交点区域，再回到底层做精确测试。

### 5.10 反射权重

命中后计算：

```hlsl
edgeFade
distanceFade
fresnelFade
weight = Intensity * edgeFade * distanceFade * fresnelFade
```

各权重作用：

- `edgeFade`：靠近屏幕边缘时淡出，避免反射突然断掉。
- `distanceFade`：步进越远越淡，减少远距离不稳定反射。
- `fresnelFade`：根据视线和法线角度增强掠射角反射。

最终输出：

```hlsl
return half4(SampleSource(refinedUV).rgb * weight, weight);
```

RGB 是乘过权重的反射颜色，alpha 保存权重，后面的 Balance 合成会用到 alpha。

### 5.11 Blur pass

`FragBlur` 使用 5 tap 分离高斯模糊：

```hlsl
center * 0.4026
+/- 1.3846 * 0.2442
+/- 3.2308 * 0.0545
```

C# 会先设置 `_SSRBlurRadius = (BlurRadius, 0)` 做横向，再设置 `_SSRBlurRadius = (0, BlurRadius)` 做纵向。

分离模糊比直接 2D 模糊更便宜，适合 SSR 这种屏幕空间后处理。

### 5.12 最终合成 pass

Additive：

```hlsl
return source.rgb + reflection.rgb;
```

这种方式最直观，反射直接加到画面上，容易更亮。

Balance：

```hlsl
return source.rgb * (1 - reflection.a) + reflection.rgb;
```

这种方式会根据反射 alpha 混合原图和反射，视觉上更克制，但可能让原图局部变暗。

## 6. RendererData 挂载关系

当前 `屏幕空间反射_Renderer.asset` 中挂载了两个 RendererFeature：

```yaml
m_RendererFeatures:
- ScreenSpaceAmbientOcclusion
- SSR
```

SSR 是当前自定义反射功能。SSAO 不是 SSR 必须项，但它也可能促使 URP 生成法线相关资源。SSR 自身已经请求 Depth 和 Normal，所以只要 RendererFeature 正常入队，理论上不需要依赖 SSAO。

当前 SSR feature 的 `m_Active: 1`，说明它是启用状态。

## 7. 关键实现细节和注意事项

### 7.1 为什么要复制相机颜色

SSR 需要从当前屏幕颜色中采样反射，同时最终又要写回相机颜色。如果直接读写同一个 RT，结果可能不稳定。

所以当前流程先做：

```csharp
cameraColor -> _SSRSourceCopy
```

后续所有颜色采样都从 `_SSRSourceCopy` 读取，最终才写回 `cameraColor`。

### 7.2 为什么 pass 数要检查

C# 中会按固定 index 调用 shader pass：

```csharp
GenerateHiZ = 0
Raymarching = 1
Blur = 2
Addtive = 3
Balance = 4
```

如果 shader pass 少于 5 个，Unity 会报 invalid pass index，导致效果完全不执行。当前通过：

```csharp
mMaterial.passCount >= RequiredPassCount
```

提前阻止这种情况。

### 7.3 为什么要跳过非 Game/SceneView 相机

Unity Editor 中有很多特殊相机，例如材质预览相机、Inspector preview 相机。这些相机可能没有正常的 color target 或 depth target。

当前代码只允许：

```csharp
CameraType.Game
CameraType.SceneView
```

这样可以减少编辑器预览窗口触发 SSR 后的空引用问题。

### 7.4 SSR 的天然限制

SSR 只能反射当前屏幕已经能看到的信息。以下内容无法可靠反射：

- 屏幕外物体。
- 被前景遮挡的物体。
- 天空盒或没有有效深度的区域。
- 透明物体之后的真实几何。
- 反射方向指向相机背后的信息。

所以当前实现使用边缘淡出、距离淡出和模糊来减少这些限制带来的突兀感。

## 8. 参数调节建议

| 现象 | 可调参数 |
| --- | --- |
| 没有效果或反射太弱 | 增大 `Intensity`，确认材质/场景有可反射颜色，确认 RendererData 使用当前 SSR Feature。 |
| 反射断裂明显 | 增大 `StepCount`，减小 `Stride`，增大 `Thickness`。 |
| 性能较差 | 减小 `StepCount`，增大 `Stride`，开启 `UseHiZ`，降低 `MipCount`。 |
| 自己反射自己或有黑边 | 增大 `RayBias`，适当增大 `Thickness`。 |
| 屏幕边缘反射突兀 | 增大 `EdgeFade`。 |
| 远处反射闪烁 | 增大 `DistanceFade`，适当增加 `BlurRadius`。 |
| 反射过糊 | 降低 `BlurRadius`。 |
| 反射过亮 | 降低 `Intensity` 或改用 `Balance`。 |

推荐起始值：

```text
Intensity: 0.6 - 0.9
MaxDistance: 8 - 15
Stride: 4 - 8
StepCount: 48 - 96
Thickness: 0.2 - 0.5
BinaryCount: 4 - 6
RayBias: 0.02 - 0.06
BlurRadius: 0.5 - 1.5
UseHiZ: true
MipCount: 6 - 9
```

## 9. 调试流程

如果 SSR 挂到 URP RendererData 后看不到效果，可以按这个顺序排查：

1. 确认当前相机使用的 URP RendererData 是 `屏幕空间反射_Renderer.asset`。
2. 确认 RendererData 中 `SSR` 的 `m_Active` 为 1。
3. 确认 `Hidden/SSR` shader 编译没有报错。
4. 确认 `SSR.cs` 中 `RequiredPassCount = 5`，shader 里也确实有 5 个 pass。
5. 确认 RendererData 的 `m_IntermediateTextureMode` 为 1。
6. 确认当前渲染的是 Game 或 SceneView 相机。
7. 确认场景有不透明物体写入深度。
8. 确认物体法线正常，URP 能生成 `_CameraNormalsTexture`。
9. 临时关闭 `UseHiZ`，确认基础线性 raymarch 是否能出效果。
10. 临时增大 `Intensity`，降低 `Stride`，增大 `StepCount`，确认不是参数太保守。

## 10. 当前实现总结

当前 SSR 是一个完整的 URP 屏幕空间反射后处理链路：

```text
URP opaque rendering
-> request depth and normal
-> allocate SSR temporary RTs
-> allocate and build HiZ depth pyramid
-> copy camera color
-> reconstruct position and normal in shader
-> compute reflection ray
-> screen-space DDA raymarch
-> optional HiZ acceleration
-> binary search refine hit UV
-> sample reflected color
-> fade and weight reflection
-> separable blur
-> additive or balance composite
-> write back to camera color
```

这套实现已经具备基础 SSR、屏幕光栅空间步进、二分细化、dither 降噪、HiZ 加速、模糊和最终合成。后续如果继续扩展，可以考虑加入粗糙度控制、法线厚度自适应、反射 mask、temporal accumulation、blue noise jitter、反射 fallback probe，以及按材质属性控制 SSR 强度。
